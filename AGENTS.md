# AGENTS.md

## このプロジェクトの目的

このプロジェクトは、済生会横浜市東部病院 救命救急センターにおける EHCU 入室時・転科時の院内運用を、若手医師・当直医が迷わず確認できる静的Webアプリとして実装するためのものです。

アプリ名は以下です。

EHCU Rule Navigator

---

## 最重要方針

このアプリは、医療判断そのものを自動化するものではありません。

目的は、院内運用ルールをSTEP形式で見やすく提示し、連絡先・処理順序・次に行う行動を確認しやすくすることです。

患者情報は絶対に入力させないでください。

---

## 実装方針

以下を厳守してください。

- 単一HTMLで作成する
- ファイル名は `index.html`
- HTML / CSS / JavaScript をすべて `index.html` 内に書く
- Reactは使わない
- Tailwindは使わない
- 外部ライブラリは使わない
- CDNは使わない
- localStorageは使わない
- 患者情報入力欄は作らない
- フォーム送信機能は作らない
- データ保存機能は作らない
- サーバー通信は行わない
- ブラウザで直接 `index.html` を開いて動作するようにする

---

## UI方針

スマホ利用を最優先してください。

- 画面幅 390px 前後で見やすくする
- トップ画面では「EHCU入室時フロー」と「転科時フロー」の2択にする
- 大量の選択肢だけプルダウンにする
- 少ない選択肢はボタンにする
- チェックボックスは使わない
- 判定結果を上に表示する
- 次の操作を判定結果の下に表示する
- ボタンは大きくする
- プルダウンも大きく、タップしやすくする
- 片手操作しやすくする
- 誤タップ防止のため、ボタン間隔を十分に空ける
- STEPタイトルは大きく太字にする
- 判定結果カードは目立たせる
- 実行STEPを大きく見やすく表示する
- 余計な装飾は入れない
- Apple風のシンプルで洗練されたUIにする
- 医療現場で瞬時に理解できる視認性を優先する

## 3. EHCU内転科成立時の共通STEP

EHCU内転科が成立した場合は、総合内科ルート・他科ルートを問わず、以下を表示してください。

共通STEPカードとして表示してください。  
背景は白または淡い青でよいです。  
番号付きで大きく見やすく表示してください。

1. 転科先へ依頼
2. 直接電話またはLINE WORKSで情報共有
3. 救急科で電子カルテ上の主科・主治医変更
4. 救急科で重症指示簿を消去
5. 平日日勤：転科先が一般病棟用指示簿へ書き換え
6. 夜間・休日：EHCU病棟番が一般病棟用指示簿へ書き換え
7. 平日日勤：転科先がドクターコール順序を記載
8. 夜間・休日：翌平日日勤帯に転科先がドクターコール順序を記載

注意：
共通STEPでは、平日日勤と夜間・休日で担当者が異なる項目を明確に分けて表示してください。
